Étapes d'un projet

Jour 1 - Planifier son projet

  • Réfléchir à son projet, savoir exactement ce que l'on veut
  • Analyser les besoins
  • Distinguer les différents composants du projet
  • Découper le projet en différentes tâches
  • Estimer le temps de chaque tâche
  • Répartir les tâches (échanger une tâche avec un collègue => petit travail d'équipe)
  • Si besoin d'une base de données, créer le Modèle Conceptuel de Données (MCD) avec DBDesigner ou MySQLWorkbench ou autre (la database sera créée sur un serveur distant)
  • FAIRE VALIDER PAR FORMATEUR
  • Faire le zoning (maquettes, mockups) des pages (pencilProject ou Balsamiq)
  • Organiser son code (créer fichiers et répertoires dès le début)
  • FAIRE VALIDER PAR FORMATEUR
  • Utiliser les outils nécessaires au travail en équipe (Git, partage de fichiers, etc.)

Avant de coder

  • Avoir fait toute la partie "Planifier son projet" notée au dessus
  • Avoir créé les tables du projet dans la DB d'un membre de l'équipe sur wf3.progweb.fr
  • Avoir créé le repo Git
  • Avoir créé le fichier .gitignore (ignorer nbproject, config.php, vendor, etc.)
  • Avoir créé tous les dossiers et les fichiers vides
  • Avoir déposé les répertoires et fichiers sur la branche master sur le repo "origin"
  • FAIRE VALIDER PAR FORMATEUR
  • Avoir activé sur 1 SourceTree le "GitFlow"
  • Avoir fait un "push" de la branche "develop" sur origin
  • Avoir récupéré la branche origin/develop sur chaque SourceTree
  • Avoir activé "GitFlow" sur tous les SourceTree

Jour 2/3 - Coder

  • Bien gérer les priorités (fonctionnalités avant la cosmétique)

Recommandations

  • Faire une page de contact
  • Utiliser l'organisation de fichiers du projet TOTO (afin de ne pas répéter son code)
